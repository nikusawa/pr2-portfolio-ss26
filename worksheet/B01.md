## 1. Git

### 1.1 Git Status erklären

Der Befehl git status zeigt den aktuellen Zustand der lokalen Git Repo im Branch b03.
„Changes not staged for commit" bedeutet, dass diese Dateien Git bereits bekannt sind (sie wurden früher einmal mit git add hinzugefügt), aber die aktuellen Änderungen noch nicht für den nächsten Commit vorgemerkt (gestaged) wurden:

- CONTRIBUTING.md – verändert, aber nicht gestaged
- homework/b03.md – verändert, aber nicht gestaged

„Untracked files" bedeutet, dass diese Datei Git völlig unbekannt ist – sie wurde noch nie zu Git hinzugefügt und wird von Git daher vollständig ignoriert:

- foo.java – neu, noch nie von Git erfasst

Befehlssequenz: nur foo.java committen
Da foo.java eine unversionierte Datei ist, muss sie zuerst explizit zum Staging-Bereich hinzugefügt werden. Die anderen beiden Dateien dürfen dabei nicht gestaged werden.
bashgit add foo.java
git commit -m "Add foo.java"
Warum funktioniert das?

git add foo.java fügt ausschließlich foo.java zum Staging-Bereich (Index) hinzu.
CONTRIBUTING.md und homework/b03.md bleiben unberührt im Zustand „modified, not staged".
git commit überträgt dann nur das, was im Staging-Bereich liegt – also nur foo.java.
